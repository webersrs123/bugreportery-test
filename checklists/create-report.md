# Чек-лист тестирования создания баг-репорта

## Функциональность создания бага

| ID | Проверка | Статус | Баг | Примечание |
|----|----------|--------|-----|------------|
| CREATE-001 | Создание бага с заполнением всех обязательных полей | ⬜ | – | Обязательные: ID, Название, Описание, Шаги воспроизведения, Ожидаемый результат, Фактический результат |
| CREATE-002 | Попытка сохранить баг с пустыми значениями во всех полях (обязательные и необязательные) | ⬜ | – | Ожидается, что форма не отправится, появятся сообщения об ошибках для обязательных полей |
| CREATE-003 | Попытка сохранить без ID бага | ⬜ | – | |
| CREATE-004 | Попытка сохранить без Названия бага | ⬜ | – | |
| CREATE-005 | Попытка сохранить без Описания | ⬜ | – | |
| CREATE-006 | Попытка сохранить без Шагов воспроизведения | ⬜ | – | |
| CREATE-007 | Попытка сохранить без Ожидаемого результата | ⬜ | – | |
| CREATE-008 | Попытка сохранить без Фактического результата | ⬜ | – | |
| CREATE-009 | Валидация сообщений об ошибках (понятны ли, подсвечивают ли поле?) | ⬜ | – | |
| CREATE-010 | Автогенерация ID (BUG-001, BUG-002…) — проверить уникальность | ⬜ | – | |
| CREATE-011 | ID — можно редактировать вручную? Если да — проверить дубликаты и формат | ⬜ | – | |
| CREATE-014 | Сохранение бага доступно только авторизованным пользователям | ⬜ | – | Проверить, что неавторизованный пользователь не может сохранить баг (кнопка неактивна, запрос отклонён) |

## Поля с выпадающими списками (необязательные)

| ID | Проверка | Статус | Баг | Примечание |
|----|----------|--------|-----|------------|
| CREATE-012 | Выбор значения из списков: серьёзность, приоритет, тип бага, статус | ⬜ | – | Поля не обязательны, должны работать корректно |
| CREATE-013 | Сохранение бага без заполнения этих полей | ⬜ | – | Ожидается успешное сохранение |

## Безопасность

| ID | Проверка | Статус | Баг | Примечание |
|----|----------|--------|-----|------------|
| SEC-CREATE-001 | XSS: ввод `<script>alert(1)</script>` в текстовые поля (Название, Описание, Шаги и т.д.) — не исполняется | ⬜ | – | |
| SEC-CREATE-002 | SQL-инъекция: ввод `' OR '1'='1` в текстовые поля — не нарушает запрос | ⬜ | – | |
| SEC-CREATE-003 | ID бага: попытка ввести вредоносный код (XSS, SQL) — должно экранироваться | ⬜ | – | Поле ID вводится вручную |
| SEC-CREATE-004 | Проверка, что в текстовых полях не сохраняются скрытые теги/скрипты после сохранения | ⬜ | – | После сохранения открыть баг — проверить, что код не исполняется |

## UI / UX

| ID | Проверка | Статус | Баг | Примечание |
|----|----------|--------|-----|------------|
| UI-CREATE-001 | Отображение формы на мобильных устройствах (эмуляция) — все поля видны, кнопки не перекрываются | ⬜ | – | |
| UI-CREATE-002 | Сообщения об ошибках понятны, подсвечивают проблемное поле | ⬜ | – | |
| UI-CREATE-003 | При ошибке отправки введённые данные (кроме ID, если он генерируется) сохраняются | ⬜ | – | Пароля нет, данные должны оставаться в полях |

## API (опционально)

| ID | Проверка | Статус | Баг | Примечание |
|----|----------|--------|-----|------------|
| API-CREATE-001 | POST /rest/v1/bugs — успешное создание бага, статус 201 | ⬜ | – | |
| API-CREATE-002 | POST без обязательных полей — статус 400, сообщение об ошибке | ⬜ | – | |
| API-CREATE-003 | Проверка, что ID бага уникален и соответствует формату | ⬜ | – | |

*Статус: ⬜ – не выполнено, ✅ – пройдено, ❌ – не пройдено (баг).*
