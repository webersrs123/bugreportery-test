
# Чек-лист тестирования создания баг-репорта

## Функциональность создания бага

| ID | Проверка | Статус | Баг | Примечание |
|----|----------|--------|-----|------------|
| TK-CREATE-001 | Создание бага с заполнением всех обязательных полей | ⬜ | – | Обязательные: ID, Название, Описание, Шаги воспроизведения, Ожидаемый результат, Фактический результат |
| TK-CREATE-002 | Попытка сохранить баг с пустыми значениями во всех полях | ⬜ | – | Ожидается, что форма не отправится, появятся сообщения об ошибках для обязательных полей |
| TK-CREATE-003 | Попытка сохранить без ID бага | ⬜ | – | |
| TK-CREATE-004 | Попытка сохранить без Названия бага | ⬜ | – | |
| TK-CREATE-005 | Попытка сохранить без Описания | ⬜ | – | |
| TK-CREATE-006 | Попытка сохранить без Шагов воспроизведения | ⬜ | – | |
| TK-CREATE-007 | Попытка сохранить без Ожидаемого результата | ⬜ | – | |
| TK-CREATE-008 | Попытка сохранить без Фактического результата | ⬜ | – | |
| TK-CREATE-009 | Загрузка изображения (скриншота, вложения) — успешное прикрепление, отображение после сохранения | ⬜ | – | |
| TK-CREATE-010 | Автогенерация ID (BUG-001, BUG-002…) — проверить уникальность | ⬜ | – | |
| TK-CREATE-011 | ID — можно редактировать вручную? Если да — проверить дубликаты и формат | ⬜ | – | |
| TK-CREATE-012 | Выбор значения из списков: серьёзность, приоритет, тип бага, статус | ⬜ | – | Поля не обязательны, должны работать корректно |
| TK-CREATE-013 | Сохранение бага без заполнения полей со списками | ⬜ | – | Ожидается успешное сохранение |
| TK-CREATE-014 | Сохранение бага доступно только авторизованным пользователям | ⬜ | – | Если по требованиям требуется авторизация |

## Безопасность

| ID | Проверка | Статус | Баг | Примечание |
|----|----------|--------|-----|------------|
| SEC-CREATE-001 | XSS: ввод `<script>alert(1)</script>` в текстовые поля — не исполняется | ⬜ | – | |
| SEC-CREATE-002 | SQL-инъекция: ввод `' OR '1'='1` в текстовые поля — не нарушает запрос | ⬜ | – | |
| SEC-CREATE-003 | ID бага: попытка ввести вредоносный код (XSS, SQL) — должно экранироваться | ⬜ | – | |
| SEC-CREATE-004 | Проверка, что в текстовых полях не сохраняются скрытые теги/скрипты после сохранения | ⬜ | – | |
