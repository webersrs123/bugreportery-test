
## ТК-CREATE-009: Загрузка изображения (скриншота) — успешное прикрепление, отображение после сохранения

**Предусловие:**
- Открыта страница https://bugreportery.qatest.tech
- Пользователь авторизован c email test@qatest.tech
- Поле загрузки изображения активно, принимаемые форматы: .jpg, .png, .gif
- Подготовлен тестовый файл-скриншот test_screenshot.png
- Кнопка "Выбрать файлы" доступна поосле прокруткм вниз

**Окружение:** Windows 11, Chrome 146 (режим инкогнито), 1920x1080

**Дата:** 24.03.2026

**Шаги:**
1. Нажать на кнопку "Выбрать файлы"
2. В диалоге выбора файла указать test_screenshot.png и нажать «Открыть».
3. Убедиться, что рядом с полем появилось имя файла или миниатюра 
4. Заполнить остальные обязательные поля формы регистрации валидными данными.
5. Нажать кнопку "Сохранить репорт"

**Ожидаемый результат:**
1. После выбора файла отображается имя файла и превью, сообщений об ошибке нет
2. После сохранения формы данные успешно отправляются, сервер возвращает статус 200
3. В сохранённом баг-репорте присутсвует загруженное изображение отображается корректно 
4. Название изображения соответствует загруженному, изображение не повреждено
