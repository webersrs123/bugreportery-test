# Баг-репорт BUG-REG-003

## Название
При регистрации с уже существующим email форма сообщает об успехе, но в БД пользователь не добавляется.
| Проект | BugReportery |
|--------|----------------|
| Серьёзность | Major |
| Приоритет | High |
| Тип бага | Регистрация / БД |
| Статус | Открыт |

## Описание
При попытке зарегистрироваться с email, который уже существует в системе, форма отображает сообщение «Регистрация успешна! Проверьте email для подтверждения», однако в Базе данных новый пользователь не создаётся. Это нарушает ожидаемое поведение: должен быть отказ с сообщением о дубликате.

## Окружение
- **Устройство:** Desktop
- **ОС:** Windows 11 Pro
- **Браузер:** Google Chrome 146.0.7680.80
- **URL:** https://bugreportery.qatest.tech

## Шаги воспроизведения
1. Зарегистрировать пользователя с email `test@qatest.tech` (если ещё нет).
2. Повторно заполнить форму регистрации с тем же email, но с другим именем (например, `TestUser2`) и другим паролем.
3. Нажать «Зарегистрироваться».

## Фактический результат
- Появляется сообщение «Регистрация успешна! Проверьте email для подтверждения».
- В консоли запрос к Supabase завершается со статусом 200.
- В таблице `auth.users` новый пользователь не появляется.

## Ожидаемый результат
- Должна быть ошибка: «Пользователь с таким email уже зарегистрирован».
