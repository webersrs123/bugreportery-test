# Баг-репорт BUG-REG-011

## Название
В модальном окне входа отображается ложное сообщение «Вход выполнен»

| Проект | BugReportery |
|--------|----------------|
| Серьёзность | Minor |
| Приоритет | Medium |
| Тип бага | UI / Функциональность |
| Статус | Открыт |

## Описание
После успешного выхода из системы (логаута) при открытии модального окна входа поля email и пароль уже заполнены (автозаполнения браузера). Появляется красное сообщение «Вход выполнен», хотя фактически вход не происходит. Это дезориентирует пользователя.

## Окружение
- **Устройство:** Desktop
- **ОС:** Windows 11 Pro
- **Браузер:** Яндекс Браузер 25.12.4.1216 (64‑бит) / Google Chrome 146.0.7680.80 (64‑бит) Режим инкогнито
- **URL:** https://bugreportery.qatest.tech

## Шаги воспроизведения
1. Авторизоваться под пользователем (например, test@qatest.tech / *****).
2. Нажать кнопку «Выход».
3. Убедиться, что сессия завершена.
4. Нажать кнопку «Вход» в хедере.
5. Обратить внимание на поля и надпмись «Вход выполнен» в модальном окне.

## Фактический результат
- Поля email и пароль уже заполнены (автозаполнения браузера)(например, email: test@qatest.tech, пароль: ••••••••).
- В модальном окне входа отображантся красное сообщение «Вход выполнен», но фактически пользователь вход не выполнял.


## Ожидаемый результат
-Поля email и пароль уже заполнены (автозаполнения браузера) (например, email: test@qatest.tech, пароль: ••••••••).
- Не должно появляться сообщение об успешном входе, если вход непроизводился.
