# Баг-репорт BUG-010

## Название
UI/UX / Сессия не сохраняется после закрытия браузера

| Проект | BugReportery |
|--------|----------------|
| Серьёзность | Major |
| Приоритет | High |
| Тип бага | UI/UX / Функциональность |
| Статус | Открыт |

## Описание
После авторизации, закрытия браузера и повторного открытия сайта пользователь не остаётся авторизованным. Сессия не восстанавливается, поля входа пусты, требуется повторный ввод логина и пароля.

## Окружение
- **Устройство:** Desktop
- **ОС:** Windows 11 Pro
- **Браузер:** Google Chrome 146.0.7680.80
- **URL:** https://bugreportery.qatest.tech

## Шаги воспроизведения
1. Открыть страницу https://bugreportery.qatest.tech (в обычном режиме)
2. Нажать кновку "Вход"
3. Авторизоваться (email: test@qatest.tech, пароль: *****).
2. Закрыть браузер полностью
3. Открыть браузер заново.
4. Перейти на страницу https://bugreportery.qatest.tech.

## Фактический результат
- Пользователь не авторизован.
- В хедере отображаются кнопки «Вход» и «Регистрация».
- Поле Email пустое. В поле пароль отображается ******
- Для доступа к функциям требуется повторный вход.

## Ожидаемый результат
Пользователь должен оставаться авторизованным после перезапуска браузера: в хедере отображается имя пользователя и «Выход», доступ к личному кабинету сохраняется.
