# Баг-репорт BUG-008

## Название
UI / Авторизация — после выхода из системы кнопка «Выход» остаётся на странице

| Проект | BugReportery |
|--------|----------------|
| Серьёзность | Minor |
| Приоритет | Medium |
| Тип бага | UI/UX |
| Статус | Исправлен |

## Описание
После успешного выхода из системы (логаута) кнопка «Выход» в хедере не исчезает и остаётся активной. Пользователь остаётся на главной странице, но визуально интерфейс не обновляется, создавая впечатление, что выход не произошёл. При этом запрос к Supabase возвращает статус 204, сессия действительно завершена.

## Окружение
- **Устройство:** Desktop
- **ОС:** Windows 11 Pro
- **Браузер:** Яндекс Браузер 25.12.4.1216 (64‑бит) / Google Chrome 146.0.7680.80 (64‑бит) Режим инкогнито
- **URL:** https://bugreportery.qatest.tech

## Шаги воспроизведения
1. Открыть страницу https://bugreportery.qatest.tech
2. Нажать кновку "Вход"
3. Авторизоваться (email: test@qatest.tech, пароль: *****).
4. Нажать кнопку «Выход».
5. Обратить внимание на хедер страницы.

## Фактический результат
- Кнопка «Выход» остаётся на месте, не заменяется на кнопки «Вход» / «Регистрация».
- В консоли запрос `/auth/v1/logout` возвращает статус 204 (успех).

## Ожидаемый результат
После выхода интерфейс должен обновиться: кнопка «Выход» должна исчезнуть, а вместо неё появиться кнопки «Вход» и «Регистрация».
