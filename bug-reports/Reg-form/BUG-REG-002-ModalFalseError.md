# Баг-репорт BUG-REG-002

## Название
При успешной регистрации пользователь видит сообщение «Database error saving new user», в консоли ошибка 500, хотя пользователь создаётся

| Проект | BugReportery |
|--------|----------------|
| Серьёзность | Major |
| Приоритет | High |
| Тип бага | Регистрация / UI |
| Статус | Исправлен |

## Описание
При заполнении формы регистрации валидными данными и нажатии кнопки «Зарегистрироваться» пользователь создаётся в Supabase (таблица `auth.users`), но в модальном окне отображается красное сообщение «Database error saving new user», а в консоли браузера появляется ошибка 500 Internal Server Error от Supabase. Это вводит пользователя в заблуждение и нарушает ожидаемый UX.

## Окружение
- **Устройство:** Desktop
- **ОС:** Windows 11 Pro
- **Браузер:** Google Chrome 146.0.7680.80 (64‑бит)
- **URL:** https://bugreportery.qatest.tech

## Шаги воспроизведения
1. Открыть страницу https://bugreportery.qatest.tech
2. В хедере нажать «Регистрация»
3. Заполнить поля:
   - Имя пользователя: `TestUser`
   - Email: `test@qatest.tech`
   - Пароль: `*****`
4. Нажать «Зарегистрироваться»

## Фактический результат
- В модальном окне появляется сообщение красным: «Database error saving new user».
- В консоли ошибка:
POST https://qfurbgyuahfrwsrpyzzy.supabase.co/auth/v1/signup 500 (Internal Server Error)
AuthApiError: Database error saving new user

text
- При этом в Supabase (таблица `auth.users`) пользователь успешно создаётся.

## Ожидаемый результат
- После успешной регистрации пользователь должен видеть сообщение об успехе (или перенаправление).
- Консоль не должна содержать ошибок.
- Пользователь не должен видеть сообщение об ошибке, если операция выполнена.

## Дополнительная информация
- Проблема не всегда воспроизводится: иногда первый запрос падает с 500, но повторный (или фоновая обработка) завершается успешно.
- Предположительно, проблема на стороне Supabase (триггеры, политики) или в логике фронтенда, который не дожидается повторной попытки.

**Причина:** Проблема была на стороне Supabase (временный сбой built-in email сервиса)

**Решение:**
- Подтверждение email работает стабильно, пользователь создаётся корректно.
- Перенос сервиса подтверждения Email на Resend.com
